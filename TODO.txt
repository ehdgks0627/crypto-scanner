PQC 전환 자동화 시스템 · 구현 체크리스트
프로젝트: 양자택일 (Quantum-Either)
근거: 최종 발표 자료 (MainDeck.html)
체크 기준: [ ] 미구현 / [~] 부분 구현 / [x] 구현 완료 / [-] 향후 확장 (의도적 미포함)
최근 점검: 2026-05-16, 현재 main 코드 기준
점검 메모:
  · Host Agent 최소 구현을 반영했다. 시스템 CA, 인증서 파일, 개인키 파일, SSH 설정, 애플리케이션 TLS 설정을 수집한다.
  · 개인키는 원문을 저장하지 않고 path/algorithm/key_size/fingerprint/in_use/dormant 메타데이터만 저장한다.
  · Discovery Agent TLS probe가 서버에서 수락되는 TLS 1.2/1.3 cipher suite 후보를 열거해 자산화한다.
  · Host Agent는 SSH 기반 원격 배포 스크립트로 systemd 서비스 설치까지 자동화한다.
  · LLM 정성 분석은 mock-rulebook 기반 자산별 큐 작업으로 실행 가능하다.
  · LLM 호출 모듈은 OpenAI 호환 Chat Completions provider를 환경변수 기반으로 호출하고, 미구성 시 mock-rulebook을 유지한다.
  · LLM 정성 분석 prompt는 자산 메타데이터, 컨텍스트, risk score를 구조화해 주입한다.
  · LLM operational context는 파일 경로, 연결 서비스, 데이터 분류, 통신 노출 범위를 정규화한다.
  · LLM 자유 텍스트 응답은 JSON 객체를 추출해 QualitativeAssessment 필드로 검증/정규화한다.
  · LLM 정성 분석 응답 실패/타임아웃은 휴리스틱 결과로 폴백하고 실패 사유를 prompt_payload에 기록한다.
  · LLM 정성 분석은 prompt 입력 해시를 저장해 동일 입력 재실행 시 기존 결과를 캐시로 재사용한다.
  · LLM 정성 분석은 DHS Q1 자산 가치(asset_value)를 외부 노출과 비즈니스 중요도 기준으로 구조화한다.
  · LLM 정성 분석은 DHS Q2 보호 정보(protected_information)를 데이터 분류와 기밀성 기준으로 구조화한다.
  · LLM 정성 분석은 DHS Q3 통신 범위(communication_scope)를 내부 전용/외부 양방향 기준으로 구조화한다.
  · LLM 정성 분석은 DHS Q4 공유 수준(sharing_level)을 3rd-party/파트너/공개 연동 기준으로 구조화한다.
  · LLM 정성 분석은 DHS Q5 핵심 인프라(critical_infrastructure)를 DB/인증/결제/KMS/게이트웨이 의존도 기준으로 구조화한다.
  · LLM 정성 분석은 DHS Q6 보호 기간(protection_duration)을 보존 기간과 HNDL 노출 기준으로 구조화한다.
  · DHS 6기준은 HNDL 보호 기간에 가장 큰 가중치를 두고 0~10 score_10 및 P1/P2/P3로 합산한다.
  · 자산 목록과 위험평가 표는 DHS 점수(score_10)와 우선순위(P1/P2/P3)를 컬럼으로 표시한다.
  · 마이그레이션 알고리즘 매핑은 migration_engine/mapping_rules.json 설정 파일로 외부화했다.
  · 마이그레이션 계획은 asset_purpose로 키 교환/디지털 서명/키 합의/장기 서명 용도를 API와 대시보드에 표시한다.
  · RSA 자산은 용도별로 분기해 key/key_agreement/protocol은 ML-KEM-768, certificate는 ML-DSA-65 경로를 권고한다.
  · ECDSA P-256 계열은 ECDSA-P-256/prime256v1/secp256r1 표기를 ML-DSA-65 경로로 정규 매핑한다.
  · X25519/Curve25519, ECP-256, DH group14/MODP-2048 표기는 ML-KEM-768 키 합의 전환 경로로 정규 매핑한다.
  · archive/audit/timestamp 등 장기 보존용 서명 컨텍스트는 SLH-DSA-SHA2 계열로 분기한다.
  · CBOM export는 마이그레이션 계획 annotation과 자산별 migration.* 속성을 함께 첨부한다.
  · 대시보드 전환 계획 화면은 발표 흐름에 맞춰 Review Targets 명칭으로 노출한다.
  · 가용성 검사는 TLS 핸드셰이크 성공 횟수에서 success rate를 정규화하고 대시보드에 표시한다.
  · 가용성 검사는 SSH 대상의 프로토콜을 보존하고 SSH 핸드셰이크 성공률을 프로토콜별 summary로 집계한다.
  · 가용성 검사는 IKE/IPsec 협상 성공 횟수에서 negotiation success rate를 산출하고 프로토콜별 summary로 집계한다.
  · 가용성 검사는 프로토콜별 response code와 failure reason을 보존하고 summary에 집계한다.
  · baseline 스냅샷이 있는 가용성 검사는 latency 전/후 p95와 delta를 summary와 대시보드에 표시한다.
  · baseline 스냅샷이 있는 가용성 검사는 throughput_rps 전/후 처리량과 delta를 summary와 대시보드에 표시한다.
  · 가용성 검사 대시보드는 자산별 baseline/current 처리량 비교 그래프를 표시한다.
  · 가용성 검사는 legacy_tls12 등 클라이언트 프로파일별 상호운용성 결과를 보존하고 대시보드에 표시한다.
  · 가용성 검사는 프로토콜/클라이언트/응답 코드/실패 사유 단위 failure path를 요약하고 대시보드에 보고한다.
  · post_migration 가용성 검사 실행은 전환 전 CBOM 스냅샷을 baseline_snapshot_id로 자동 연결한다.
  · post_migration 가용성 검사 실행은 전환 후 CBOM 스냅샷을 post_migration_snapshot_id와 snapshot summary migration 메타데이터로 연결한다.
  · CBOM diff는 자산 누락과 알고리즘 다운그레이드를 regressions로 감지하고 스냅샷 비교 화면에 표시한다.
  · 대시보드는 POST /dashboard/demo-seed를 통해 데모용 테스트베드 시드 데이터를 즉시 로드할 수 있다.
  · 대시보드는 최신 CBOM 스냅샷 기준 스캔당 발견 암호자산 KPI를 계산해 표시한다.
  · 대시보드는 최신 CBOM 스냅샷 기준 양자 취약 암호자산 KPI를 계산해 표시한다.
  · pqc-tls 테스트베드는 ML-KEM/ML-DSA readiness endpoint를 노출하고 Discovery/Scanner가 이를 PQC 자산으로 수집한다.
  · 네트워크 스캔과 Host Agent가 같은 인증서를 발견하면 fingerprint 기반으로 단일 자산으로 병합한다.
  · Discovery Agent, 대시보드, CBOM 스냅샷, 마이그레이션 추천, 가용성 검사 UI는 코드 기준으로 반영한다.
  · KPI 숫자, 외부 도구 비교, 인용 자료는 별도 실측/사실 확인 전까지 완료로 처리하지 않는다.

    [x] 1.1.2  STARTTLS 핸드셰이크 (SMTP/IMAP/POP3 등)
    [x] 1.1.3  SSH 호스트 키 수집 (algo + fingerprint)

    [x] 1.2.1  시스템 신뢰 저장소(trust store) 스캔
              (Linux: /etc/ssl/certs, Windows: cert store, macOS: keychain 등)
    [x] 1.2.2  파일시스템에서 인증서 파일 탐지 (.pem .crt .cer .p12 .pfx 등)

    [x] 1.2.7  Host Agent가 원격 호스트에 배포 가능

# KPI Evidence

이 문서는 발표용 KPI 중 실측값이 아닌 비교 기준선을 과장 없이 설명하기 위한 근거를 정리한다.

## 15.1 Before: 수동 grep 기준

`Before · 수동 grep 기준 — 수일 ~ 수주`는 자동화 전 환경에서 사람이 직접 암호 자산 목록을 만드는 경우의 정성 비교 기준이다. 이 값은 실제 벤치마크 시간이 아니라, 현재 데모 시드 범위를 사람이 수동으로 점검할 때 필요한 작업량을 보수적으로 산정한 것이다.

구조화된 산정표는 `docs/kpi/manual-grep-baseline.json`에 둔다. 현재 기준 범위는 다음과 같다.

| 항목 | 값 |
| --- | ---: |
| 암호 자산 | 67개 |
| 발견 엔드포인트 | 33개 |
| 내부 점검 후보 호스트 | 10개 |
| 스캔 대상 | 31개 |

수동 작업에는 대상 목록 확인, 접속 권한 준비, `grep`/`find` 기반 파일 탐색, `openssl x509`/`ssh-keygen` 기반 인증서와 키 파싱, STARTTLS·SSH·TLS별 프로토콜 확인, 중복 제거, CBOM 형태 기록, 검토와 인수인계가 포함된다.

산정 결과는 최소 34.45시간, 최대 72.25시간이다. 하루 8시간 기준 약 4.31~9.03 인일이며, 실제 조직에서는 접속 권한, 검토 대기, 운영 담당자 확인 시간이 끼기 때문에 발표에서는 `수일 ~ 수주`라는 정성 표현을 사용한다.

발표 문구는 다음 수준으로 제한한다.

> 같은 범위를 사람이 수동 grep과 명령행 도구로 확인하면 작업 자체만 약 4~9 인일이 필요하고, 접근 권한과 검토 대기까지 포함하면 수일에서 수주까지 늘어날 수 있습니다.

주의할 점은 이 값이 자동화 도구의 실측 성능과 직접 비교되는 벤치마크가 아니라는 것이다. 자동화 이후의 `수 분` 또는 `10분 이내` 주장은 별도 실행 시간 측정값으로만 말해야 한다.
